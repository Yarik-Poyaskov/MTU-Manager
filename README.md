# 🛠️ MTU Manager

## 📋 Что это такое

**MTU Manager** — это PowerShell-скрипт с графическим интерфейсом (WinForms), который позволяет:

- просматривать все активные сетевые адаптеры и их текущий MTU
- вручную задавать нужное значение MTU и применять его
- проверять текущий MTU пинг-тестом с флагом DF (Don't Fragment)
- **автоматически находить максимальный рабочий MTU** через бинарный поиск
- указывать произвольный хост для проверки (по умолчанию `8.8.8.8`)

Полезно при настройке VPN, туннелей (WireGuard, OpenVPN, GRE), а также при устранении проблем с фрагментацией пакетов.

## ⚙️ Возможности

### 📡 Просмотр адаптеров
Отображает все активные (`Up`) сетевые адаптеры с текущим значением MTU. Список обновляется автоматически после применения изменений.

### ✏️ Применение MTU
Устанавливает MTU через `netsh`, затем выполняет переподключение адаптера (Disable → Enable) для применения настроек без перезагрузки ПК. Поддерживается диапазон **576–9000**.

### 🏓 Тест Ping
Отправляет ICMP-пакет с флагом **Don't Fragment** заданного размера на указанный хост. Позволяет убедиться, что пакеты с текущим MTU проходят без фрагментации.

### 🔍 Автопоиск максимального MTU
Использует **бинарный поиск** в диапазоне 576–1500 байт:
1. Проверяет верхнюю границу (1500) — если проходит, сразу готово
2. Проверяет нижнюю границу (576) — если не проходит, сигнализирует об отсутствии связи
3. Итеративно сужает диапазон за **~10 шагов** вместо 924
4. Найденное значение автоматически подставляется в поле MTU

Весь процесс отображается в цветном логе в реальном времени.

### 🌐 Произвольный хост
По умолчанию все проверки идут на `8.8.8.8` (Google DNS). Можно указать любой хост: IP-адрес или доменное имя — например, адрес корпоративного шлюза или VPN-сервера.

## 🚀 Запуск

### Быстрый старт (рекомендуется)

Запустите PowerShell **от имени администратора** и выполните:

```powershell
powershell -ExecutionPolicy Bypass -File MTU_Manager.ps1
```

> ⚠️ Права администратора обязательны — изменение MTU и управление адаптерами требуют повышенных привилегий.

### Альтернатива — разрешить скрипты глобально

```powershell
Set-ExecutionPolicy RemoteSigned -Scope CurrentUser
```

После этого можно запускать двойным кликом (если установлена ассоциация `.ps1` → PowerShell).

## 🔧 Типичные сценарии использования

**VPN / туннели** — при использовании WireGuard, OpenVPN или IPsec часто нужно снизить MTU (обычно до 1380–1420), чтобы избежать фрагментации туннельных пакетов.

**Корпоративные сети** — некоторые коммутаторы и маршрутизаторы не поддерживают Jumbo Frames или ограничивают MTU. Автопоиск быстро определяет рабочий максимум.

**Проблемы с загрузкой страниц** — симптомы: сайты частично не загружаются, SSH/RDP работает, а браузер — нет. Часто причина именно в MTU.

---

## 📝 Примечания

- После применения MTU адаптер переподключается (~5 секунд). Сетевые соединения временно прерываются.
- Значение MTU сохраняется в реестре (`store=persistent`) и переживает перезагрузку.
- Поиск максимального MTU не изменяет настройки адаптера — только определяет оптимальное значение.
